ASTC Format & Void-Extent Specification

16-Byte File Header

Every .astc file begins with a 16-byte header containing metadata:

  • Bytes 0-3: Magic 0x5CA1AB1E
  • Byte 4: Block Width (X)
  • Byte 5: Block Height (Y)
  • Byte 6: Block Depth (Z)
  • Bytes 7-9: Image Width (24-bit)
  • Bytes 10-12: Image Height (24-bit)
  • Bytes 13-15: Image Depth (24-bit)

128-Bit Block Layout

ASTC compresses textures into fixed-size 128-bit (16 bytes) blocks. The number of pixels inside each block depends on the block footprint (e.g., 4x4 = 16 pixels, 12x12 = 144 pixels).

This yields highly flexible bitrates ranging from 8.0 bits per pixel (bpp) down to 0.89 bpp, allowing developers to balance quality and memory usage.

Void-Extent Block Mode

The Void-Extent mode is a special, fully compliant ASTC block mode designed to represent constant-color regions efficiently. It is encoded as:

  • Bits 0-8: 0x1FC (Signature)
  • Bits 9-11: Coordinate Flags
  • Bits 12-63: Reserved (All 1s)
  • Bits 64-79: Red (16-bit UNORM)
  • Bits 80-95: Green (16-bit UNORM)
  • Bits 96-111: Blue (16-bit UNORM)
  • Bits 112-127: Alpha (16-bit UNORM)
